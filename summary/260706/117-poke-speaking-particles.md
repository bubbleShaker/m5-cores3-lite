# #117 ポケモンシーンへ発話中パーティクルを配線（全シーン共通化の完了）

## 背景
発話中パーティクル（実音声連動の楕円リング）は #109 で羊に導入、#116/#118 で純粋層の楕円化と統合層
`drawSpeakingParticles(now, cx, cy, rx, ry, bg, excl[], exclN)` への切り出しを済ませ、羊・宝石には配線済み。
残るポケモンシーンだけ未対応だった（#117 の残タスク）。

## ポケ固有の構造差
ポケには発話が2フェーズある。
1. **鳴き声(cry)＋揺れ(jiggle)** — `pokeOnTap` 内のブロッキングループ（main loop は回らない）
2. **解説(commentary TTS)** — `speakTts` が非同期再生で即 return し、以後 `pokeUpdate` が毎フレーム呼ばれる

`pokeUpdate` は従来空（静止画）で per-frame フックが無く、揺れは onTap ループという二重構造が難所だった。

## 変更（配線のみ・新規ロジックゼロ）
- 共有ヘルパー `drawPokeSpeakingParticles(now)` を追加。jiggle ループ（①）と `pokeUpdate`（②）の双方から
  同一ジオメトリで呼ぶ。統合層は非発話時に早期 return するので静止画表示中はほぼ無コスト。
- ジオメトリは宝石ハローと同形：中心＝スプライト中心(160,66)、rx=104。
- 除外矩形＝jiggle 帯（揺れるスプライトを内包）＋上部ヘッダ帯。背景は kColBg（黒・単色）。
- `pokeOnTap` 先頭に `resetSpeakingParticles()`（宝石 gemOnTap と同じ作法）。

## レビュー指摘と対応（reviewer サブエージェント）
- 🔴 **must**: 純粋層 `particle_ring` は基準半径に脈動 `pulse`(最大11)＋粒半径(最大4)を足す。当初 `ry=52`
  だと大音量ピーク時に下端が `cy(66)+52+15=133` まで届き、名前(上端≈124)に粒子が乗る。`pokeUpdate` は
  カードを再描画しないため、黒消去で**次タップまで文字が欠けたまま**になる欠陥。
  → `kPokeRingRy` を **52→40** に下げて解消（worst-case 下端 121 < 名前上端 124）。左右ロブは y≈cy で
  rx 依存のため見た目はほぼ不変。
- 🟡 should / 🟢 nit（本PRスコープ外・別 Issue 化）：
  - シーン切替時に `Speaker.stop()` を挟まないため、長尺解説の再生中に切替えると遷移先が他人の声で光る
    （羊・宝石共通の既存挙動だが長尺解説のポケで顕在化しやすい）。
  - 宝石にも同種の脈動オーバーラン余地（`kGemCy=62`＋`ry=52`）が潜在。
