# #132 起動時メニュー画面＋音声読み上げ ON/OFF トグル

## 背景 / 課題
- 音声読み上げ（TTS 解説・鳴き声・対話）を導入して以降、宝石図鑑・ポケモン図鑑などの表示が、中継サーバーの有無に関わらず遅くなっていた。音声のための HTTP がブロッキングで走り、相手不在時は接続待ちで固まるため。
- シーン遷移が「長押しでローテーション巡回」のみで、サーバー未起動のまま羊/ポケモン等に不意に入ると必ず失敗し不便だった。

## 対応内容

### 1. 音声読み上げ ON/OFF（既定 OFF）
- グローバル `g_voiceEnabled`（既定 `false`）を追加。
- **音声を出す/対話する全経路の入口でガード**して、OFF 時は HTTP を一切張らずに即 return する:
  - `speakTts` / `prefetchTts` / `playBleat` / `speakCry` / `recordAndTranscribe`（/stt）/ `fetchChatReply`（/chat）
- **ポケモンの info/sprite（`pokeLoad`）は遮断しない**。絵の表示に必須で「音声導入前から必要」な通信のため。これにより OFF 時は読み上げ導入前の表示速度に戻る。
- ON 時の表示高速化は別 Issue で対応予定。

### 2. 起動時メニュー画面（シーン選択のハブ）
- 起動直後の最初の画面をシーン選択メニューにした。縦リストで **先頭に「音声: ON/OFF」トグル**、続けて全シーン（ひつじ / アート / 宝石図鑑 / ポケモン図鑑 / 音量調整 / 声の選択 / スタックチャン）を表示。
- 操作: **上半分タップでカーソル上、下半分で下、長押しで決定**。先頭行の決定は音声トグル（メニューに留まる）、シーン行の決定はそのシーンへ入場。
- **シーン中の長押しでメニューへ戻る**。従来のローテーション巡回は廃止。
- 「最初に音声を選べる」「メニューからも切替」「最初の画面がページ選択」を、音声トグルをメニュー先頭行に統合することで1画面で実現（専用の起動画面を増やさない）。

## 実装メモ
- カーソル移動は純粋ロジック `menu_move(current, count, delta)`（端でクランプ・範囲外正規化・count<=0 安全値）に切り出し、`test/test_menu` で境界を網羅（native 全 174 テスト green）。
- `SceneDef` に `name` を追加し、シーン追加＝配列に1行足すだけでメニューにも出る（開放閉鎖）。
- メニュー行が画面をはみ出さないことを `static_assert` でコンパイル時保証（シーン追加で溢れたらビルドで止まる）。
- ループはメニュー/シーンの二モード制。メニュー中は早期 return しシーン `update` の裏描画を防止。シーン→メニュー復帰は `Speaker.stop()`→`resetSpeakingParticles()`→`exit()`→`menuEnter()` の順。
